## Testing Patterns

### End-to-End (Batch)

**Approach:** JCL submits a batch job that:
1. Deletes prior output datasets (IEFBR14 step)
2. Runs the pricer (HOSOP210) with test input files
3. Reads the output RATEFILE via SAS
4. Produces a formatted report (`PROC PRINT`) with key fields

**Input files (referenced in JCL, not in repo):**
- `HOSPV210.PRODBILL.INPUT` — test bill records
- `HOSPV210.PROVFILE` — provider reference data
- `HOSPV210.CBSA2021` — CBSA wage index data
- `HOSPV160.MSAH050` — legacy MSA wage index data

**Output validation:**
- SAS report includes: PROVIDER, FROM_DT, ADMT_DT, PAY_AMT_TOT, RTC, HIGH_DAYS, LOW_DAYS
- Manual comparison of expected vs actual payment amounts and return codes

## Local Testing (GnuCOBOL)

**Environment:** GnuCOBOL 3.2 on Windows (`tools/gnucobol32/`)

**Test Programs (in `test/` directory):**
- `GENDATA.cbl` — Test data generator: produces `PROVFILE` (3 providers) and `BILLFILE` (45 test cases: TC01–TC41 including TC08B, TC27A/B, TC28A/B, TC29A/B)
- `HOSOP210.cbl` — Local test driver: substitutes the production CICS wrapper; opens files, loads tables, invokes HOSDR210 for each bill
- `build-and-run.bat` — Automated build + execution script (7 steps)

**Build & Run:**
```
test\build-and-run.bat
```

Steps: (1) strip IBM line numbers from sources, (2) patch COPY statement for `.cpy`, (3–6) compile HOSPR210, HOSDR210, GENDATA, HOSOP210, (7) generate data and run pricer.

**Output:** `run/RATEFILE` — priced claim records (LINE SEQUENTIAL format, ~275 bytes per record due to trailing space trimming)

**Data Files (in `run/` directory):**
- `PROVFILE` — Generated by GENDATA (3 providers: 341234 Charlotte, 341235 NYC, 341236 Abilene TX)
- `BILLFILE` — Generated by GENDATA (45 test bills)
- `CBSAFILE` — Copy of `CBSA2021` (7,478 CBSA wage index records)
- `MSAFILE` — Manual creation (1 record: MSA 6740, WI=1.0000 for pre-FY2006 cases)
- `RATEFILE` — Output from pricer execution

**Local Gate Check:**
```bash
test\build-and-run.bat
# Check RATEFILE output for expected RTC codes and payment amounts
```
